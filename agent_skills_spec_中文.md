# 智能体技能规范

技能是由说明、脚本与资源组成的文件夹，智能体可以按需发现并加载它们，以在特定任务上获得更好的表现。要使文件夹被识别为技能，必须包含 `SKILL.md` 文件。

# 技能目录结构

最小化的技能目录如下：

```
my-skill/
  - SKILL.md
```

更复杂的技能可以按需添加额外的子目录与文件。

# SKILL.md 文件

技能的“入口”是 `SKILL.md`，也是唯一必需存在的文件。该文件必须以 YAML 前言区开头，随后是普通 Markdown 内容。

## YAML 前言区

前言区包含 2 个必填属性：

- `name`
    - 技能名称，使用连字符命名（hyphen-case）
    - 仅限小写 Unicode 字母数字与连字符
    - 必须与包含该 SKILL.md 的目录名称一致
- `description`
    - 描述技能的功能以及 Claude 何时应该使用它

还可以选择性提供 3 个属性：

- `license`
    - 技能使用的许可证
    - 建议简明表述（写许可证名称或附件中的许可证文件名）
- `allowed-tools`
    - 预先批准可运行的工具列表
    - 目前仅在 Claude Code 中受支持
- `metadata`
    - 字符串键到字符串值的映射
    - 客户端可借此存储规范未定义的附加属性
    - 建议使用足够独特的键名以避免冲突

## Markdown 正文

正文不受任何限制，可根据需求撰写。
